PR_SUMMARY: |
  ## Summary
  Adds a `--conclusions` CLI flag (and matching `CONCLUSIONS` env var) to filter `ci-channel` notifications by run outcome. The channel is marketed as a CI **failure** alerting tool, but it previously forwarded every pipeline event regardless of outcome. This PR changes the default behavior to enforce strict failure-only alerting, dropping known non-failure states and in-progress noise, while maintaining an escape hatch (`--conclusions all`) for the old behavior.

  ## Key Changes
  - Changes default pipeline behavior to drop `success`, `skipped`, `neutral`, `manual`, `stale` alongside in-progress events (`requested`, `running`, `pending`, etc.).
  - Forwards `failure`, `cancelled`, `timed_out`, `action_required`, and novel/unrecognized forge outcomes by default.
  - Adds the `--conclusions` config flag with cross-forge normalization (`failed` ≡ `failure`, `canceled` ≡ `cancelled`), and supports explicit custom inclusion lists.
  - Exposes active filter status in the bootstrap startup notification banner.
  - Updates documentation (README.md, INSTALL.md, AGENTS.md, CLAUDE.md) highlighting the new flag, defaults, and the upgrade path for users.
  - Version bump to 0.6.0.
