KEY_ISSUES:
- `lib/webhook.ts` default exclusion set omits GitLab `created`. The existing codebase already recognizes this as a GitLab pipeline state (`tests/forges/gitlab.test.ts` has a “passes through created state” test), and it is non-terminal/in-progress rather than failure-like. With `conclusions: null`, `isConclusionAllowed('created', null)` currently returns `true`, so GitLab “created” pipeline webhooks will still notify by default, undermining the PR’s stated default failure-alert behavior. Add `created` to `DEFAULT_EXCLUDED_CONCLUSIONS` and add/extend tests for it. Consider auditing other GitLab non-terminal statuses as well.
- I could not complete `npm test` in this read-only/restricted sandbox: failures were from `mkdtemp` EPERM and `listen 127.0.0.1` EPERM, not from the code under review. `npx tsc --noEmit` passes.
